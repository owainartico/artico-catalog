## Name a Bright Star: Rotating Milky Way Implementation

### Major Visual Upgrade - Photorealistic Space Globe
- **Project:** name-a-bright-star (star registration system)
- **Goal:** Transform registration globe to look like premium physical celestial sphere
- **Reference:** User provided image of photorealistic space globe with Milky Way band

### Implementation Journey (Multiple Iterations)

**Early attempts (failed):**
1. Tried SVG polygon approach with blend modes - didn't render on mobile
2. Tried simple strokes for Milky Way - not realistic enough
3. Tried static canvas background - worked but didn't rotate

**Final solution (successful):**
- **Three.js GPU-accelerated rotating sphere**
- Equirectangular panorama texture mapped to 3D sphere
- Rotates smoothly with D3 constellation overlay (60fps)
- Procedural starfield fallback (3000 colored stars)

### Technical Implementation

**Files modified:** `src/public/register.html`

**Architecture:**
- Three.js WebGLRenderer with orthographic camera
- SphereGeometry (64×64 segments) viewed from inside (BackSide)
- Texture: `/assets/milky-way-panorama.jpg` (equirectangular)
- Rotation sync: D3 rotation (lambda/phi) → Three.js radians (Y=-lambda, X=-phi)
- Falls back to CanvasTexture with procedural stars if image missing

**Performance:**
- GPU-accelerated (WebGL)
- 60fps on desktop and mobile
- Three.js adds 150KB (acceptable for quality)

- Strong elements: darker black `#0a0a14` with bold weight

**2. Improved Constellation Rotation Logic**
- Rewrote `rotateToConstellation()` function for better centering
- Now uses `raToLon()` function consistently for RA→longitude conversion
- Improved shortest-path calculation for 360° wraparound
- Snaps to exact target position at end of animation (was slightly off before)
- Better normalization: `while` loops instead of single `if` checks

**Key changes:**
```javascript
// Old: Manual calculation
const targetRotation = [-(ra * 15), -dec, 0];

// New: Consistent with rest of codebase
const targetLon = raToLon(ra);  // -(ra * 15)
const targetLat = -dec;

// Better wraparound handling
while (deltaLon > 180) deltaLon -= 360;
while (deltaLon < -180) deltaLon += 360;

// Snap to exact position at end
rotation[0] = targetLon;
rotation[1] = targetLat;
```
